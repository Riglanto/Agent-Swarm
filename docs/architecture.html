Contract Details

TaskManager.solBase / OApp
▼
Planner agent contract. Inherits LayerZero OApp for cross-chain messaging. Holds ETH in escrow until task completion is verified, then releases reward to worker.
Data Structures
enum TaskStatus { Created, Dispatched, Completed, Paid }

struct Task {
  uint256 id;
  string description;
  uint256 reward;
  address creator;
  address worker;
  bytes32 resultHash;
  TaskStatus status;
  uint32 workerChainEid;
}
Functions
FunctionDescription
createTask(description)Creates task with msg.value ETH locked as reward. Returns taskId.
dispatchTask(taskId, worker, dstEid, options)Sends task payload to WorkerNode on destination chain via _lzSend. Requires msg.value for LZ fee.
_lzReceive(...)Internal callback: receives result hash and worker address from WorkerNode. Sets status to Completed.
releaseReward(taskId)Transfers locked ETH to worker after task completed. Only callable by task creator.
quoteDispatch(...)View function to estimate LayerZero messaging fee for dispatch.
Events
  • TaskCreated(taskId, description, reward)
  • TaskDispatched(taskId, dstEid, guid)
  • ResultReceived(taskId, resultHash, worker)
  • RewardReleased(taskId, worker, reward)
WorkerNode.solArbitrum / OApp
▼
Worker agent contract on the destination chain. Receives dispatched tasks via LayerZero, stores them, and provides submitResult() for workers to send results back.
Data Structures
struct ReceivedTask {
  uint256 taskId;
  string description;
  uint256 reward;
  address assignedWorker;
  bool completed;
}
Functions
FunctionDescription
_lzReceive(...)Internal callback: decodes task payload from TaskManager and stores in receivedTasks mapping.
submitResult(taskId, resultHash, srcEid, options)Worker submits result hash and sends it back to TaskManager via _lzSend. Requires msg.value for LZ fee.
getTaskCount()Returns the number of tasks received by this node.
Events
  • TaskReceived(taskId, description, worker)
  • ResultSubmitted(taskId, resultHash)
WorkerRegistry.solAny Chain
▼
On-chain directory of available workers. Workers self-register with a capability string and their chain EID. The Planner agent queries this to discover and select the best workers for each subtask.
Data Structures
struct Worker {
  address addr;
  string capability;
  uint32 chainEid;
  bool active;
}
Functions
FunctionDescription
register(capability, chainEid)Worker self-registers with capability string and chain EID. Adds to workerList if new.
deactivate()Worker sets themselves as inactive. They remain in the list but are excluded from queries.
getWorker(addr)Returns Worker struct for a specific address.
getWorkerCount()Returns total number of registered workers.
findWorkers(capability)Returns all active workers matching a capability string.
Events
  • WorkerRegistered(worker, capability, chainEid)
  • WorkerDeactivated(worker)
