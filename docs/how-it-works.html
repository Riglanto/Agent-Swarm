Full Task Flow

Press play to watch the entire lifecycle unfold.

1Task Submitted
User submits "Analyze wallet 0xABC" with 0.01 ETH reward.
TaskManager.createTask("Analyze wallet 0xABC")
→ taskId: 0  |  reward: 0.01 ETH  |  status: Created
2Planner Decomposes
Planner breaks task into two subtasks based on capabilities needed.
subtask[0]: "Count Base transactions" → needs: data-analysis
subtask[1]: "Review contract interactions" → needs: code-review
3Workers Discovered
WorkerRegistry queried for matching capability and chain location.
registry.findWorkers("data-analysis") → [Worker #1, Arbitrum]
registry.findWorkers("code-review") → [Worker #2, Arbitrum]
4Tasks Dispatched Cross-Chain
Subtasks sent to Workers on Arbitrum via LayerZero V2 _lzSend.
TaskManager._lzSend(dstEid: 40231, payload)
→ packet flies Base ⟶ LayerZero ⟶ Arbitrum
WorkerNode._lzReceive() stores task, emits TaskReceived
5Workers Execute
Each worker processes its subtask off-chain, then submits result hash on-chain.
Worker #1: "42 transactions found on Base for 0xABC"
Worker #2: "3 contract interactions: Uniswap, Aave, Lido"
6Results Return
Result hashes sent back to TaskManager on Base via LayerZero.
WorkerNode.submitResult(taskId, resultHash, srcEid: 30184)
→ packet flies Arbitrum ⟶ LayerZero ⟶ Base
TaskManager._lzReceive() → status: Completed
7Rewards Released
Task creator verifies results and releases ETH to workers.
TaskManager.releaseReward(0) → 0.005 ETH → Worker #1
TaskManager.releaseReward(1) → 0.005 ETH → Worker #2
status: Paid ✓

Contract Architecture

TaskManager.sol OApp
▼
Planner contract on Base. Creates tasks with ETH escrow, dispatches to workers cross-chain via LayerZero V2, receives result hashes, and releases rewards.
Functions
  • createTask(description) payable → taskId
  • dispatchTask(taskId, worker, dstEid, options) payable
  • releaseReward(taskId)
  • quoteDispatch(taskId, dstEid, worker, options) view
  • _lzReceive(origin, guid, message, ...) internal
Events
  • TaskCreated(taskId, description, reward)
  • TaskDispatched(taskId, dstEid, guid)
  • ResultReceived(taskId, resultHash, worker)
  • RewardReleased(taskId, worker, reward)
State
  • tasks mapping — id, description, reward, creator, worker, resultHash, status, workerChainEid
  • nextTaskId counter
WorkerNode.sol OApp
▼
Worker contract on Arbitrum. Receives dispatched tasks via LayerZero, stores them locally, and submits result hashes back cross-chain.
Functions
  • submitResult(taskId, resultHash, srcEid, options) payable
  • getTaskCount() view → uint256
  • _lzReceive(origin, guid, message, ...) internal
Events
  • TaskReceived(taskId, description, worker)
  • ResultSubmitted(taskId, resultHash)
WorkerRegistry.sol Registry
▼
On-chain directory of workers. Workers self-register with their capability string and chain EID. The Planner queries this to discover and select workers.
Functions
  • register(capability, chainEid)
  • deactivate()
  • getWorker(addr) view → Worker
  • getWorkerCount() view → uint256
  • findWorkers(capability) view → Worker[]
Events
  • WorkerRegistered(worker, capability, chainEid)
  • WorkerDeactivated(worker)
